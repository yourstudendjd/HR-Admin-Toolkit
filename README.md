# HR 管理工具集

> 一个基于 **PyQt5** 的桌面工具，持续迭代中。当前包含四个功能模块，覆盖 HR 日常工作中的考勤、宿舍、出入库和对账数据处理。

[![Python](https://img.shields.io/badge/Python-3.x-blue)](https://www.python.org/)
[![PyQt5](https://img.shields.io/badge/PyQt5-5.15+-green)](https://www.riverbankcomputing.com/software/pyqt/)
[![License](https://img.shields.io/badge/license-MIT-lightgrey)](LICENSE)

## 功能模块总览

| 模块 | 图标 | 功能说明 |
|------|------|----------|
| 考勤汇总 | 📋 | 读取打卡 XLSX → 每日汇总表 + 迟到判定 + 乐捐金额明细 |
| 宿舍分摊 | 🏠 | 读取宿舍费用 XLSX → 按人按天分摊水电费 + 住宿费 |
| 出入库流水 | 📦 | 读取出入库流水 XLSX → 按部门/存货清洗分类汇总 |
| 对账数据清洗 | 🧹 | 客户对账单批量去重分类 → 一个工作簿多工作表输出 |

- Python 3.8 或更高版本

## 模块详解

### 📋 考勤汇总

**功能**：将每日打卡记录汇总为按人按日的统计表，支持迟到判定和乐捐金额计算。

**输入**：打卡记录 XLSX 文件（可选班次表 XLSX 用于迟到判定）

**输出**：汇总表 XLSX（包含每日出勤状态、迟到分钟数、乐捐金额）

**乐捐规则**：

| 迟到分钟 | 乐捐金额 |
|---------|---------|
| 1-10 分钟 | 15 元 |
| 11-30 分钟 | 30 元 |
| 31-59 分钟 | 100 元 |
| ≥60 分钟 | 300 元 |

**操作流程**：
1. 拖拽或选择打卡记录文件
2. （可选）拖拽或选择班次表文件以启用迟到判定
3. 勾选"启用迟到检测"
4. 点击「开始处理考勤」
5. 查看日志中的统计摘要，或打开输出文件夹

**排除人员**：可在代码中修改 `DEFAULT_EXCLUDE_NAMES` 列表，排除不需要统计的人员（如长期离职人员）。

---

### 🏠 宿舍分摊

**功能**：将宿舍水电费按居住人天比例分摊，住宿费按固定标准计算。

**输入**：宿舍费用 XLSX 文件（含房间号、费用类型、金额等）

**输出**：分摊明细表 XLSX（每人每项费用明细 + 汇总）

**分摊规则**：
- **住宿费**：50 元/天，按当月实际居住天数比例计算
- **水电费**：按房间汇总后，按每人居住天数占房间总人天的比例分摊
- **同名员工**：如员工换宿舍出现在多个房间，住宿费仅在天数最多的房间显示，水电费在各房间均计算

**操作流程**：
1. 选择宿舍费用文件
2. 设置计费年月（程序会尝试从文件名自动识别）
3. （可选）勾选"将上月最后一天的截止日期视为仍在住"
4. 点击「开始分摊计算」
5. 查看结果摘要（涉及住客数、房间数、总金额）

### 📦 出入库流水

**功能**：对材料出库流水进行分类清洗，按经手人映射部门，按存货名称汇总。

**输入**：出入库流水 XLSX 文件

**输出**：清洗后的分类汇总表 XLSX

**部门映射规则**：

| 经手人 | 映射部门 |
|--------|---------|
| 彭建森 | VCP |
| 罗德浮 | VCP |
| 曹鹏 | 蚀刻 |
| 邹柳珍 | 曝光 |
| 刘志军 | 品质 |
| 其他 | 其他 |

**操作流程**：
1. 选择出入库流水文件
2. 选择输出路径（默认与输入文件同目录）
3. 点击「开始清洗」
4. 查看各部门的物料出库金额汇总

### 🧹 对账数据清洗

**功能**：批量处理多张客户对账单 Excel 文件，对每个文件独立进行数据清洗和去重，最后合并为一个工作簿（每文件一个工作表）。

#### 核心去重规则

| 组合条件 | 是否合并 |
|---------|---------|
| 客户型号 + 加工类型 + 加工要求 **三者都相同** | ✅ 合并为 1 条 |
| 客户型号 + 加工类型 + 加工要求 相同，**规格不同** | ✅ 合并（规格不参与去重） |
| 加工类型不同 | ❌ 不合并 |
| 加工要求不同 | ❌ 不合并 |

#### 单价冲突处理策略

当相同组合出现不同单价时，可选择以下策略：

| 策略 | 说明 |
|------|------|
| 取首次出现的值 | 保留该组合第一条记录的单价（推荐） |
| 取平均值 | 取该组合所有单价的算术平均值 |
| 取最大值 | 取该组合中的最高单价 |
| 提示冲突报错 | 发现冲突时中断处理，提示用户手动处理 |

#### 自动列名模糊匹配

每个文件的列名可能不同（如 "客户产品型号" vs "客户型号"），程序会自动识别常见列名变体：

| 目标字段 | 匹配关键词 |
|---------|-----------|
| 客户型号 | 客户型号、客户产品型号、产品型号、型号、品名、客户品号 |
| 单价 | 单价、单价（元/个）、单价(元)、价格、unit_price |
| 加工类型 | 加工类型、加工工艺、工艺、加工、类型 |
| 规格 | 规格、客户规格、产品规格、尺寸、大小 |
| 加工要求 | 加工要求、要求、加工标准、工艺要求 |

#### 自动处理

- **表头行自动检测**：每个文件独立扫描前 25 行，自动识别表头所在行
- **合并单元格处理**：使用 `header=None` 读取避免列名错位
- **单位说明行过滤**：自动过滤表头后的单位行（如 "mm"、"铜箔" 等）
- **日期格式化**：Excel 日期序列号自动转为 `YYYY-MM-DD` 短日期格式
- **隐藏不需要的列**：自动隐藏规格列、送货单号、总价等不需要输出的列

#### 输出格式

输出为一个 Excel 工作簿，包含：
- `_处理汇总` 工作表：列出每个文件的处理明细（原始行数、清洗后行数、合并数、状态）
- 各文件的独立工作表：以原文件名命名，内容为清洗后的数据

#### 操作流程

1. 点击 **「+ 添加文件」**，选择多个对账文件（支持 Ctrl+ 多选）
2. 每个文件自动检测表头行 + 模糊匹配列名
3. 在文件列表中查看匹配状态（已匹配 5/5 / 部分匹配 / 未匹配）
4. 选中列表中的文件，在下方 **「选中文件的列映射」** 区域可手动调整个别字段的映射
5. 设置全局 **单价冲突策略**
6. 点击 **「开始批量对账清洗」**
7. 查看处理摘要，打开输出文件夹获取结果

## 开发指南

### 添加新处理模块

1. 在 `main.py` 中新建 `XxxWorker(QThread)` 工作线程类
2. 新建 `XxxTab(QWidget)` 标签页类，遵循现有 UI 风格
3. 在 `MainWindow.init_ui()` 中注册标签页
4. 在 `data_cleaner_processor.py` 中提取核心逻辑为独立类

### 修改去重规则

编辑 `data_cleaner_processor.py` 中的 `make_key` 函数：

```python
def make_key(row):
    return (
        str(row[col_model]),    # 客户型号
        str(row[col_type]),     # 加工类型
        str(row[col_req]),      # 加工要求
        # 如需加入规格，取消下面注释：
        # str(row[col_spec]),   # 规格
    )
```

### 修改列名匹配关键词

编辑 `data_cleaner_processor.py` 中的 `FIELD_KEYWORDS` 字典：

```python
FIELD_KEYWORDS = {
    '客户型号': ['客户型号', '客户产品型号', '产品型号', ...],
    '单价':     ['单价', '单价（元/个）', '价格', ...],
    ...
}
```

## 常见问题

**Q: 列名匹配不上怎么办？**
A: 在文件列表中选中该文件，在下方「选中文件的列映射」区域手动选择对应的列即可。

**Q: 不同文件的表头行不一样怎么办？**
A: 程序会自动检测每个文件的表头行。如果自动检测不准，可以在添加文件后手动调整「表头行号」下拉框的值。

**Q: 输出文件中有些列不想显示怎么办？**
A: 程序已自动隐藏规格、送货单号、总价等列。如需调整，编辑 `data_cleaner_processor.py` 中的 `drop_keywords` 列表。

**Q: 可以同时处理多少文件？**
A: 理论上无上限，实际取决于文件大小和内存。建议单次处理不超过 20 个文件。
