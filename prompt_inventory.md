# 项目：出入库流水自动化清洗与分类汇总工具

请帮我生成一个**完整的、单文件的 Python 脚本**，命名为 `process_inventory.py`，能够自动处理出入库流水 Excel 文件。

## 输入文件格式

输入文件为 `出入库流水.xlsx`，结构如下：

**前6行**：标题/元信息（需要跳过）
- 第1-2行：标题"出入库流水"
- 第3-5行：空行或备注
- 第6行：期间和单据日期信息

**第7行**：真正的列头（关键）：

**第8行起**：数据行
**最后一行**：合计行（包含"合计："字样）

## 核心功能要求

### 1. 数据读取
- 自动跳过前6行
- 正确识别第7行为列头
- 自动识别并跳过最后的合计行（通过检测第一列是否包含"合计"）
- 处理可能存在的空单元格（数值列填0，文本列填空字符串）

### 2. 数据筛选
- **只保留** `单据类型` 为 `"材料出库单"` 的记录
- 排除：进货单、销货单等其他类型

### 3. 部门映射（核心）
根据 `经手人` 列的值，映射到部门代号，并**新增或替换 `部门` 列**：

| 经手人 | 映射后的部门 |
|--------|-------------|
| 彭建森 | VCP |
| 罗德浮 | VCP |
| 曹鹏 | 蚀刻 |
| 邹柳珍 | 曝光 |
| 刘志军 | 品质 |
| 其他任何值（包括空值） | 其他 |

映射逻辑：如果经手人匹配上述任一姓名，则部门=对应值；否则部门="其他"。

### 4. 数据排序与分类汇总
对筛选后的数据：
- 先按 `存货` 升序排序
- 再按 `部门` 升序排序
- 然后对每个 `存货` 分组，计算：
  - 该存货的**出库数量合计**
  - 该存货的**出库金额合计**

### 5. 生成多工作表 Excel 输出文件
输出文件名格式：`出入库流水_清洗后_YYYYMMDD_HHMMSS.xlsx`

**工作表1：`汇总`**
- 使用数据透视表格式，展示按存货汇总的出库金额
- 列：`存货` | `出库金额合计`
- 底部显示总计行

**工作表2-5：分部门明细**
分别创建：`VCP`、`曝光`、`蚀刻`、`品质` 四个工作表

每个工作表包含以下列（顺序固定）：
- 只包含该部门的材料出库记录
- 按 `存货` 排序
- 如果部门无数据，在工作表第一行写入"无数据"
- 在数据末尾添加该部门的汇总行（出库数量合计、出库金额合计）
- 从 N1 单元格开始，汇总当前工作表同一存货的总数量、平均单价、总出库金额

**工作表6：`其他`**
- 存放部门="其他"的所有记录（经手人未匹配或为空）
- 列结构与分部门明细相同

### 6. 控制台日志输出

### 7. 错误处理
- 如果输入文件不存在，提示用户并退出
- 如果关键列（如"经手人"、"存货"）找不到，打印可用列名并退出
- 数值转换失败时，尝试清理（去除货币符号、逗号等）后转换，仍失败则填0

## 技术约束
- 仅使用 Python 标准库 + pandas + openpyxl
- 单文件实现，不要拆分成多个模块
- 使用 `if __name__ == "__main__":` 作为入口

## 使用方法
脚本支持两种运行方式：

**方式1：命令行参数**
```bash
python process_inventory.py --input 出入库流水.xlsx --output 自定义输出名.xlsx
```

**方式2：无参数默认值**
```bash
python process_inventory.py
```
自动使用当前目录下的 `出入库流水.xlsx` 作为输入。
